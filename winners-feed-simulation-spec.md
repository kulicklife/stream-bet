# Лента победителей: симуляция активности игроков

Версия 2.0, 2026-06-11. Модуль готов и лежит в `sim/`:

- `sim/winners-feed-generator.ts` - генератор (zero-dependency TypeScript, вся математика внутри)
- `sim/winners-feed-generator.test.ts` - автотесты инвариантов: `npx tsx sim/winners-feed-generator.test.ts`
- `sim/demo.ts` - прогон сессии в консоль глазами: `npx tsx sim/demo.ts [seed]`

## Зачем

Реальные ставки игроков с 1x в реальном времени забирать пока не можем, а мертвый
экран без активности - продуктовое опущение. Модуль генерирует правдоподобный поток:
лента победителей, банк раунда, касса за день. Когда 1x даст реальный фид - модуль
заменяется другим источником тех же самых событий, фронт не меняется.

## Как с ним работать (3 вызова)

```ts
const sim = createSimulation({ sessionId, date, currency: 'USD' });

sim.prematch();
// один раз при старте эфира -> 10-15 записей для тикера, пока раундов еще не было

sim.generateRound({ round, winner, voided? });
// после каждого round_result -> { activity, winners, kassaToday }
//   activity: { round, betCount, bank }  - банк раунда на экран результата
//   winners:  [{ nickMasked, payout, marketTag, delayMs }] - в тикер,
//             каждый показывать через свой delayMs (не пачкой!)
//   kassaToday - в подвал

sim.restoreKassa(value);
// после рестарта сервиса: касса хранится у вас (Redis/БД, ключ - дата)
```

Что остается на команде: подключить к шине событий game loop, персистить кассу,
не эмитить победителей по уже сгенерированному раунду при round_aborted
(generateRound с voided:true вернет пустой список сам).

## Что за модель внутри (по-человечески)

1. Объем ставок растет по ходу сессии (зрители подтягиваются), с шумом - соседние
   раунды всегда разные. Всплески на раундах 1-2 и 28-29 (длинные окна ставок),
   легкий спад в самом конце. После раунда с крупным выигрышем объем проседает.
2. Большинство выигрышей мелкие, хвост тяжелый. Киты появляются случайно
   (к концу сессии чаще): иногда ноль за раунд, иногда пять.
3. У игроков маркетов больше, чем у блогеров: РАУНД, СЕРИЯ, ТОТАЛ, ФОРА, ЭКСПРЕСС.
   В конце четных раундов чаще закрываются серии, в конце таймов (27, 54) -
   крупные расчеты тоталов и экспрессов. Все вероятностно, жестких правил нет -
   зритель не должен вычислить паттерн.
4. Коэффициенты наружу не показываем (у игроков они динамичные) - только тег маркета.
5. Связь с реальным матчем: если в раунде сломалась серия побед, победителей меньше,
   но выплаты крупнее (зрители видели победу андердога).
6. Ники: пул на 400 штук фиксируется на дату - меняются между днями, повторяются
   внутри дня. 20 "завсегдатаев" мелькают часто, 15 китов - повторяющиеся персонажи.
   Маскируем как "KO***A". Одна маска максимум 2 раза за раунд и не подряд.
7. Арифметика не ловится на лжи: сумма показанных выплат всегда меньше банка,
   один выигрыш не больше 40% банка, касса дня = сумма банков, выплаты некруглые,
   мелочь меньше ~$11 не показываем.
8. Детерминизм: одинаковые sessionId+date (или seed) дают идентичный поток -
   для QA и реплеев. Прогон `demo.ts` с разными seed - разные сессии.

## Важное вне кода

- Комплаенс: выигрыши фиктивные, витрина - оператора. Нужен sign-off 1x до прода.
- Валюты: USD и RUB поддержаны (RUB-шкалы свои), валюта - из конфига сессии.
- Тюнинг: peakBetsPerRound и прочее - параметры createSimulation, крутится без
  изменения логики. После первых живых прогонов подстроим.
