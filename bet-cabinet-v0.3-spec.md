# Кабинет ставок блогера v0.3 - спека для фронтенда

Дата: 2026-06-10. Заменяет панель ставок из сборки v0.0.3 (две зеркальные колонки с 8 кнопками).
Референс-реализация: `bet-cabinet-v0.3.html` (один файл, токены совпадают с demo-v0.2.2, тема boxing-bout).
Дев-фикстура: открыть с `?state=series` - покажет конфиг серии у синего и активную серию с прогрессом у красного.

## Главная идея

У блогера всегда собрана дефолтная ставка: сумма $10 + "Победа <своя сторона>" ×1.90.
Чтобы поставить - одно нажатие СТАВКА. Селекторы суммы и исхода нужны только если хочет изменить.

## ВАЖНО: семантика серии изменилась

В demo v0.2.x серия означала "у кого больше побед в диапазоне [from, to]", коэффициент считался
формулой `1.85 + sqrt(n) * 0.65` (отсюда x3.30 в текущей сборке). Это устарело.

Актуальная семантика (Требования v3, раздел 1.6.1): серия - это N побед ПОДРЯД одной стороны.
Одно поражение внутри серии = ставка проиграна сразу. Коэффициенты фиксированные, не считаются:

| Длина серии | Коэф  | Маржа  |
|-------------|-------|--------|
| 1 (= раунд) | 1.90  | 5.00%  |
| 2           | 3.69  | 7.84%  |
| 3           | 7.08  | 11.53% |
| 4           | 13.60 | 15.07% |
| 5           | 26.09 | 18.46% |

Решение зафиксировано 2026-06-10. Значение 1.80 из сборки v0.0.3 и pari-mutuel формулы
из odds.ts старой демки не использовать.

## Элементы кабинета (на каждого блогера)

1. Шапка: метка угла, имя, БАЛАНС (живой, обновляется после каждого раунда).
2. Селектор суммы: $1 / $3 / $5 / $10 / $15 / $20 / $50 / $100. Номиналы - это валюта
   сессии, пока USD. Суммы больше лимита показываются, но задизейблены.
3. Селектор исхода: Победа Синий ×1.90, Победа Красный ×1.90, Серия Синий, Серия Красный.
   Коэффициент текущего выбора виден прямо на кнопке.
4. Конфиг серии (виден только при выбранной серии): длина 2-5 (сегмент-кнопки).
   Начало серии НЕ выбирается - всегда текущий раунд. UI показывает "РАУНДЫ N-M".
   Длина 1 недоступна - это дубль маркета "Победа".
5. СТАВКА - размещает ставку. Активна только в фазе BETTING OPEN.
   Вне фазы - задизейблена с текстом "СТАВКИ ЗАКРЫТЫ".
6. АВТО - тумблер автоповтора. Только для ставок на раунд (для серии задизейблен с
   подписью "ТОЛЬКО РАУНД"). При включении: в начале каждого BETTING OPEN автоматически
   ставится текущая сумма на текущий исход. Если сумма превышает лимит - АВТО выключается,
   строка лимита мигает красным.
7. Строка лимита: "ЛИМИТ СТАВКИ 10% ОТ БАЛАНСА - МАКС $X". X = floor(balance * 0.1).
8. Полоса активной серии (видна пока есть pending-серия): сторона, диапазон раундов,
   пипсы прогресса, "ВЫИГРАНО N ИЗ M", "ЖДЕМ РАУНД K".
9. МОИ СТАВКИ: список ставок, новые сверху. Pending - желтая метка и потенциальная выплата
   (stake ×odds -> payout), won - зеленая (+$X), lost - красная зачеркнутая (-$X).

## Правила (бизнес-логика)

Зафиксированные ответы на вопросы команды (2026-06-10):

- Одна ставка на исход раунда за раунд. Это автоматически закрывает вопрос
  взаимоисключающих ставок: поставить и на П1, и на П2 в одном раунде нельзя физически.
  После размещения кнопка показывает "СТАВКА СДЕЛАНА".
- Раунд + серия одновременно - можно. Это разные маркеты. Стороны могут не совпадать
  (раунд на П1 + серия на П2 разрешено): это хедж, а не абсурд, гарантированного
  выигрыша не дает (маржа), а драмы добавляет. Взаимоисключение действует только
  внутри маркета "исход раунда".
- Серия включает текущий раунд: стартует с раунда, в фазе приема ставок которого
  размещена. Выбирается только длина (2-5), стартовый раунд не выбирается.
- Одна активная (нерассчитанная) серия на блогера - в любой момент времени, не только
  в рамках фазы. Пока серия живет, новые серии недоступны, блогер видит прогресс
  "выиграно N из M". Итого максимум в одной фазе приема ставок: 1 ставка на раунд
  + 1 серия (если нет активной).
- Ставка списывается с баланса в момент размещения. Отмена невозможна.
- Лимит: одна ставка не больше 10% текущего баланса. Проверка на момент размещения.
  Пропорциональный лимит для серий (чем длиннее - тем меньше макс) в MVP НЕ вводим:
  риск вниз уже ограничен 10%, а резкий рост баланса при заходе серии - это
  джекпот-момент, нужный шоу. Если тестовые прогоны покажут проблему - follow-up
  через cap на потенциальную выплату (например, stake * odds <= 100% баланса),
  а не через таблицу лимитов.
- Конец серии не может выйти за последний раунд сессии (длины, выходящие за предел,
  дизейблятся).
- Сеттлмент раунда: round-ставки этого раунда выигрывают/проигрывают по победителю.
  Серии: победа стороны - wins+1, поражение - ставка lost немедленно.
  wins == длина - won, выплата stake * odds (округление до $1).
- Abort сессии / void раунда: все pending-ставки возвращаются на баланс (stake назад,
  без выигрыша). В референсе не реализовано - реализовать по событию round_voided.
- Блогер может ставить и на себя, и на противника - и раунд, и серию.
- В песочнице блогера активен только его кабинет. У зрителя на Live X оба кабинета
  видны как часть шоу, но некликабельны.

## Что показывает каждая фаза

- BETTING OPEN: все контролы активны, таймер фазы сверху.
- CARD SELECTION / RESULT: СТАВКА задизейблена, селекторы можно оставить активными
  (выбор не ставит деньги), баланс и список ставок обновляются на RESULT.
